# Projektstruktur

- `classes/` enthält ausschließlich Klassen, jeweils eine Klasse pro Datei.
- `js/` enthält Konfiguration, Levelaufbau, DOM-UI und kleine Hilfsfunktionen.
- `data/` enthält ausschließlich Daten im JSON-Format.
- `img/` und `audio/` enthalten Produktionsassets.
- `styles/` enthält aufgeteilte CSS-Dateien; `style.css` ist der zentrale Einstieg.
- `script.js` startet das Spiel und enthält keine Gameplaylogik.
- `docs/` enthält Planung, Design und Assetnachweise.

Dateinamen bleiben englisch und verwenden `kebab-case`.
